# i18n — locales, tooling, and style rules

## Locale set

| Locale | Status | Notes |
|---|---|---|
| `en` | Source of truth | All other bundles are keyed against en.json |
| `ru` | Native (hand-curated) | Owner is a native Russian speaker; reviews translations directly |
| `fr` | Placeholder | Byte-identical copy of en.json until promoted |
| `de` | Placeholder | " |
| `es` | Placeholder | " |
| `it` | Placeholder | " |
| `pl` | Placeholder | " |
| `pt` | Placeholder | " |

Two SOURCE OF TRUTH constants govern the locale set:

- `mindreset-app/i18n/routing.ts` — `locales` (the full URL-routable set)
- `mindreset-app/components/LanguagePicker.tsx` — `NATIVE_CONTENT_LOCALES`
  (the subset that's been promoted to hand-curated content)

The i18n tooling duplicates these constants (since `.mjs` files can't
import TS); the comments at the top of each `.mjs` file note this.

## Tooling

All in `mindreset-app/i18n-tools/`:

| File | Purpose |
|---|---|
| `sync-placeholders.mjs` | Copies en.json bytes into each placeholder bundle. Skips native locales. Idempotent. |
| `check-bundle-parity.mjs` | Deep-walks all 8 bundles; checks key paths, array lengths, ICU placeholder sets. Exits 1 on drift. |
| `translate-missing.mjs` | Anthropic-backed translation skeleton. `--dry-run` by default; refuses `--write` to native locales. NOT in `package.json` scripts — invoked manually pre-launch. |
| `translate-prompt.md` | System prompt for `translate-missing.mjs`. Contains the full style guide, glossary, locked feminine/formal rules, and locale-specific punctuation. |
| `pre-commit-hook.sh` | Optional client-side hook that runs `i18n:check` before commits touching `messages/*.json`. Not installed on Julia's Windows machine — Vercel prebuild covers it. |

## package.json scripts

```json
"prebuild": "npm run i18n:check",
"i18n:sync":  "node i18n-tools/sync-placeholders.mjs",
"i18n:check": "node i18n-tools/check-bundle-parity.mjs",
"i18n":       "npm run i18n:sync && npm run i18n:check"
```

`prebuild` wires the parity check into Vercel's deploy gate. A drift in
any bundle blocks deploy.

## Workflow when adding/changing copy

1. Edit `messages/en.json` (source of truth).
2. Edit `messages/ru.json` if the change is content the owner wants
   hand-curated for Russian-speaking users. Follow the style rules in
   `translate-prompt.md`.
3. Run `npm run i18n:sync` to propagate en.json to the 6 placeholder
   bundles.
4. Run `npm run i18n:check` (or rely on the prebuild gate).
5. Commit.

For pre-launch native translation passes:

```bash
node i18n-tools/translate-missing.mjs --locale=fr   # dry-run, prints diff
# owner reviews diff
node i18n-tools/translate-missing.mjs --locale=fr --write
```

The script refuses `--write` to native locales (en, ru) to prevent
overwriting hand-curated bundles.

## RU style — locked rules

These rules are enforced by the owner during native-quality review; the
translator model is briefed on them via `translate-prompt.md`.

- **Formal Вы** is the default.
- **Informal ты** is allowed at trauma-soft moments only — e.g.
  `Screening.yellowCta = "Когда будешь готова"`. Treat ты as an
  intentional rhetorical choice, not a sloppy register switch.
- **Feminine grammatical forms** are canonical. Female-presenting reader
  is the assumed default in the absence of a gender signal.
- **Use ё** consistently (no е-substitution for ё in formal copy).
- **Quote marks**: « » for RU/FR/ES/IT/PT; „ " for DE/PL.
- **Locked translations** for product/brand terms — see the glossary in
  `translate-prompt.md`.
- **Currency stays Latin** — £14.99 is "£14.99" in RU, not transliterated.

## Pre-existing constraints carried forward

- Phase 3c safety keyword scanner is **EN-only** — RU phrases not yet
  added. See `docs/carry-forward.md`.
- Auth pages (sign-in, sign-up) and legal pages (terms, privacy) are
  **EN-only**. They render via Clerk-hosted UI and raw JSX respectively;
  i18n wiring deferred to a later phase.
- `/account` has no language toggle in the URL bar — the LanguagePicker
  is in the Footer. See `docs/carry-forward.md` "Language toggle missing
  on /account".

## What's enforced vs. what's trusted

Enforced mechanically by tooling:
- Key-path parity across all 8 bundles
- ICU placeholder sets match (e.g. `{name}` exists in every bundle that
  has the source key)
- Array lengths match
- Vercel prebuild fails on drift

Trusted (owner-reviewed, not mechanically checked):
- RU style adherence (formal/feminine/ё)
- Quote-mark consistency
- Locked-glossary adherence
- "Trauma-informed" usage on payment surfaces (see
  `product/philosophy.md`)
